---
name: 审查变更
description: 沿标准轴和规格轴审查自某个固定点以来的变更
---

审查自某个固定点以来的变更，沿两条平行轴进行评估：

1. **标准** — 差异是否符合仓库的编码规范？
2. **规格** — 差异是否忠实地实现了原始 issue/PRD？

## 工作流程

1. **确定基准点。** 询问用户"审查自哪个提交/分支以来的变更？"如果仓库有默认分支（如 `main`），则以此为默认值。
2. **收集差异。** 运行 `git diff <base>...HEAD` 获取自基准点以来的完整差异。
3. **逐轴审查。** 对每条轴，生成一份结构化报告：

### 标准轴

- 代码风格是否一致？
- 是否存在未使用的导入/变量？
- 错误处理是否遵循项目约定？
- 命名是否清晰且符合约定？
- 是否有遗漏的测试？

### 规格轴

- 变更是否完整实现了 issue/PRD 中描述的行为？
- 是否有边缘情况被遗漏？
- UI/UX 变更是否与设计稿匹配？
- 是否存在偏离规格的"过度工程"？

4. **综合报告。** 将两条轴的发现合并为一份可操作的审查报告，按严重程度排序（阻断 > 警告 > 建议）。

## 输出格式

```
## 审查报告：自 <base> 以来的变更

### 标准

- [阻断/警告/建议] 描述
- ...

### 规格

- [阻断/警告/建议] 描述
- ...

### 总结

整体评估和推荐的下一步行动。
```
