---
name: 碎片挖掘
description: 通过追问式对话挖掘写作碎片并追加到文档中
---

一场追问式对话，挖掘你脑中的碎片——异质的写作素材——并将它们追加到同一篇文档中，作为未来文章的原始素材。

## 工作流程

1. **确定目标文档。** 询问用户要将碎片追加到哪个文件。如果文件不存在，创建它。如果文件已存在，读取现有内容以了解已有哪些素材。

2. **开始挖掘。** 提出一个开放式问题，引发写作碎片。好问题：
   - "关于这个主题，你目前最强烈的直觉是什么？"
   - "有什么反直觉的观察你想分享？"
   - "如果只能传达一个观点，你会说什么？"
   - "什么让你对这个主题感到沮丧？"
   - "有没有一个隐喻能捕捉你的核心论点？"

3. **追问。** 不要满足于表面的回答。追问以挖掘更深层的内容：
   - "你能展开说说吗？"
   - "为什么是那个而不是其他？"
   - "那让你有什么感觉？"
   - "举个例子？"

4. **捕获碎片。** 当回答中出现了有表现力的片段——一个生动的隐喻、一个尖锐的观察、一段叙事——将其提取为碎片并追加到目标文档。碎片应保留说话者的声音——不要润色或"改善"措辞。

5. **继续或结束。** 提出另一个问题，或询问用户是否想停止。目标是收集 5-10 个碎片后再结束。

## 碎片格式

每个碎片以 markdown 二级标题和简短标签追加到文档中：

```markdown
## 碎片：关于 X 的隐喻

[原始措辞，保留说话者的声音]
```

## 原则

- 碎片是异质的——它们不需要彼此一致或对齐。矛盾是好的。
- 保留原始声音。不要将口语化的表达"翻译"为正式书面语。
- 目标是原始素材，而非成品。碎片将在后续由 `writing-shape` 技能塑造为文章。
