# RESOURCES.md 格式

`RESOURCES.md` 是本主题的精选可信来源集合。讲解器的知识应从这里提取，而不是来自参数化猜测。智慧来自这里列出的社区。

## 结构

```md
# {主题} 资源

## 知识

- [书籍：《力量训练的科学与实践》— Zatsiorsky & Kraemer](https://example.com)
  关于编程和适应的基础文本。用于：周期化、恢复、强度区间相关的任何内容。
- [文章："我应该训练多少？" — Greg Nuckols (Stronger By Science)](https://example.com)
  基于证据的容量里程碑综述。用于：每个肌肉群的每周组数目标。

## 智慧（社区）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比的 subreddit，有反对伪科学的审核。用于：计划评审、平台期排障。
- 线下：{健身房名称} 周二力量课
  用于：举重动作的实时教练反馈。
```

## 规则

- **仅高可信度来源。** 优先选择一手来源、公认专家、同行评审作品和有强审核的社区。如果一个资源是披着教育外衣的营销，不要列入。
- **为每条添加注释。** 三个月后一个裸链接毫无用处。加一行：它涵盖什么，什么时候该用它。
- **按知识/智慧分组。** 与 [SKILL.md](./SKILL.md) 中的哲学对应。一个资源只出现在一个组中也可以。
- **明确揭示缺口。** 如果使命需要的某个领域没有好的资源，写一个 `## Gaps` 章节列出缺少什么。这驱动未来搜索。
- **果断修剪。** 被证明是错误、浅薄或偏离使命的资源应该移除，不要埋没。五个精炼来源胜过三十个平庸来源。
- **记录社区偏好。** 如果用户选择不加入社区，在这里注明，这样未来会话不会反复提议。
