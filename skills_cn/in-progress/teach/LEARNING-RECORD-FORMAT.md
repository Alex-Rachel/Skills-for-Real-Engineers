# 学习记录格式

学习记录存放在 `./learning-records/` 中，使用顺序编号：`0001-slug.md`、`0002-slug.md` 等。惰性创建目录——只在写第一条记录时创建。

它们是教学版的 ADR：记录非显而易见的教训、关键洞察和声明的先验知识，这些将引导未来的会话。它们用于计算最近发展区。

## 模板

```md
# {所学或确立内容的简短标题}

{1-3 句话：学到了什么（或确立了什么先验知识），以及为什么它对未来会话重要。}
```

这就是全部格式。一条学习记录可以只有一段话。价值在于记录_这现在是已知的_以及_为什么它改变了下一步教什么_——而不是填写章节。

## 可选章节

只在它们增加真正价值时才包含。大多数记录不需要它们。

- **Status** 前置信息（`active | superseded by LR-NNNN`）——当早期理解后来证明是错误的并被替换时有用的。
- **Evidence（证据）** — 用户如何展示了理解（回答了一个问题、完成了一个练习、引用了先前经验）。当该主张可能被重新审视时有用。
- **Implications（含义）** — 这对未来会话解锁或排除了什么。当非显而易见时值得记录。

## 编号

扫描 `./learning-records/` 找到最高现有编号，然后加一。

## 何时写学习记录

当以下任一情况为真时写一条：

1. **用户展示了对非平凡事物的真正理解** — 不仅仅是接触，而是他们能正确使用概念的证据。这为下一步教什么设定了新的底线。
2. **用户披露了先验知识** — "我已经知道 X。" 记录下来，这样未来会话不会再教它。同时记录声明的_深度_。
3. **纠正了一个误解** — 用户之前相信错误的东西，现在明白了原因。这些是高价值的：它们预测相关主题的未来绊脚石。
4. **使命因学习而转变** — 用户发现他们关心的是与最初想法不同的东西。交叉链接到 [[MISSION.md]] 并更新它。

### 什么_不_符合条件

- 仅仅覆盖过的材料。覆盖不等于学习。等待证据。
- 已经在 [[GLOSSARY.md]] 中作为术语定义简洁记录的任何东西。不要重复。
- 按会话的活动日志。学习记录不是日志——它们是决策级别的洞察。

## 取代

当后来的记录与早期记录矛盾时（用户的理解加深或纠正了），将旧记录标记为 `Status: superseded by LR-NNNN`，而不是删除它。理解如何演变的历史本身就是有用的信号。
