# 领域术语表

CONTEXT.md 描述了代码库的核心领域概念。技能使用此术语表来：

- 用项目语言命名测试
- 用项目语言命名接口
- 生成与领域匹配的代码
- 审查变更是否正确建模领域

## 格式

```markdown
# 项目名称

## 领域概念

- **概念 A** — 概念 A 在此项目中的含义
- **概念 B** — 概念 B 在此项目中的含义
- **概念 C** — 概念 C 在此项目中的含义

## 关系

- 概念 A 包含多个概念 B
- 概念 B 引用概念 C
```

## 原则

- **术语必须与代码匹配。** 如果 CONTEXT.md 说"Order"，代码中就应该有 `Order` 类型。如果不匹配，要么文档错了，要么代码错了——必须解决。
- **面向领域，而非实现。** 概念应来自问题域，而非解决方案域。"Order"是领域概念。"OrderRepository"是实现细节——它不属于这里。
- **保持更新。** CONTEXT.md 是活文档。当代码引入新概念时，CONTEXT.md 必须更新。
